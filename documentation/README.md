# Документация YandexGuessr

## Обзор проекта

`yandexguessr` это браузерная игра в духе GeoGuessr. Игрок видит панораму Яндекса, ставит точку на мини-карте и получает очки в зависимости от расстояния до реальной локации.

Проект сфокусирован на Республике Саха (Якутия) и сейчас поддерживает два игровых режима:

- `YAKUTSK`: точки только в пределах Якутска
- `SAKHA`: точки по территории Республики Саха с упором на заранее заданные населенные районы

Приложение сделано как компактный клиентский React-проект на Vite и использует Yandex Maps JavaScript API для:

- поиска и рендеринга панорам
- геокодирования ближайших объектов
- отображения интерактивной мини-карты

## Цели проекта

- дать простой и быстрый географический квиз по Якутии
- использовать реальные панорамы Яндекса вместо статичных изображений
- отбирать раунды в населенных местах, чтобы игра оставалась интересной
- поддерживать интерфейс на русском и саха языках

## Технологический стек

- `React 19`
- `TypeScript`
- `Vite 5`
- `Yandex Maps JS API 2.1`
- `Tailwind CSS` через CDN в `index.html`
- `Nginx` в production Docker-образе

## Как работает приложение

1. Приложение получает API-ключ Yandex Maps.
2. Скрипт Yandex Maps динамически подключается в браузере.
3. Игрок выбирает режим игры.
4. Приложение генерирует кандидатную координату внутри выбранной территории.
5. Для этой точки ищется доступная панорама.
6. Панорама дополнительно проверяется: она должна оставаться внутри нужных границ и находиться рядом с населенным местом.
7. Игрок смотрит панораму и ставит точку на мини-карте.
8. Приложение считает расстояние и переводит его в очки.
9. После 5 раундов показывается итоговая статистика и появляется возможность поделиться результатом.

## Точки входа

- `index.tsx`: монтирует React-приложение в `#root`
- `App.tsx`: главный контейнер игры и оркестрация состояния
- `components/StreetView.tsx`: обертка над плеером панорамы
- `components/MiniMap.tsx`: карта для выбора ответа и показа результата
- `lib/yandexMaps.ts`: загрузка Yandex API и сервисные обертки
- `lib/mapUtils.ts`: география, скоринг, фильтрация и вспомогательные функции
- `lib/gameConfig.ts`: общие типы и игровые константы

## Быстрый старт

### Локальная разработка

1. Установите зависимости:

```bash
npm ci
```

2. Передайте API-ключ одним из поддерживаемых способов:

```bash
copy .env.example .env
```

Затем укажите:

```env
VITE_YANDEX_MAPS_API_KEY=your-yandex-maps-api-key
```

Либо положите ключ в `api-key.txt`.

3. Запустите dev-сервер:

```bash
npm run dev
```

4. Откройте `http://localhost:5173`

### Production-сборка

```bash
npm run build
```

Статический результат будет записан в `dist/`.

## Текущие возможности

- выбор режима игры
- игровой цикл из 5 раундов
- отображение панорам
- выбор ответа кликом по мини-карте
- начисление очков по расстоянию
- история раундов и итоговая статистика
- шаринг через Web Share API или буфер обмена
- переключение языка `RU / SAH`

## Важные ограничения

- в проекте пока нет автоматических тестов
- стили подключаются через Tailwind CDN, а не через локальный build pipeline
- приложение зависит от наличия панорам в Yandex Maps
- часть строк в исходниках выглядит как текст с проблемами кодировки и требует отдельной проверки
